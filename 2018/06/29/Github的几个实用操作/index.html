Github的几个实用操作

分享几个Github的实用操作，亲测好用。

操作一：用Github Wiki实现存储与URL访问资源

1.在Github上建一个仓库，然后将对应的Wiki项目导出到本地，如下图所示：



2.将你想要存储的文件资源放到导出的Wiki项目中，然后推到远程，这样就实现了存储，存储成功后的资源对应的URL的格式可以参考我的Wiki项目。这样你就可以稳定的访问自己的资源了，不用担心哪天出现404，除非Github挂了。建议存放一些小文件资源，因为大的资源推上去很耗时，估计是官方的限制，怕你把这里当网盘使。项目里也可以建文件夹用于分门别类，真的很方便。
